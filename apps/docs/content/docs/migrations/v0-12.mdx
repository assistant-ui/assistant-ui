## Major Architecture Change: Tap-based State Management

Version 0.13 introduces a complete rewrite of the state management system using the new `@assistant-ui/tap` library.
This brings significant performance improvements and a more consistent API, but requires substantial migration effort.

## Breaking Changes

### 1. Context Hooks Replaced with Unified State API

All individual context hooks have been replaced with a single `useAssistantState` hook and `useAssistantApi` for actions.

#### What changed

The following hooks have been removed:

**Removed Hooks:**

- `useAssistantRuntime` → Use `useAssistantApi()`
- `useThread` → Use `useAssistantState(({ thread }) => thread)`
- `useThreadRuntime` → Use `const { actions } = useAssistantApi()` with `actions.thread`
- `useMessage` → Use `useAssistantState(({ message }) => message)`
- `useMessageRuntime` → Use `const { actions } = useAssistantApi()` with `actions.message`
- `useComposer` → Use `useAssistantState(({ composer }) => composer)`
- `useComposerRuntime` → Use `const { actions } = useAssistantApi()` with `actions.composer`
- `useEditComposer` → Use `useAssistantState(({ message }) => message.composer)`
- `useThreadListItem` → Use `useAssistantState(({ threadListItem }) => threadListItem)`
- `useThreadListItemRuntime` → Use `const { actions } = useAssistantApi()` with `actions.threadListItem`
- `useMessagePart` → Use `useAssistantState(({ part }) => part)`
- `useMessagePartRuntime` → Use `const { actions } = useAssistantApi()` with `actions.part`
- `useAttachment` → Use `useAssistantState(({ attachment }) => attachment)`
- `useAttachmentRuntime` → Use `const { actions } = useAssistantApi()` with `actions.attachment`
- `useToolUIs` → Use `useAssistantState(({ toolUIs }) => toolUIs)` and `const { actions } = useAssistantApi()` with `actions.toolUIs`
- `useMessageUtils` → Use `message.isHovering` / `message.isCopied`
- `useThreadModelContext` / `useThreadModelConfig` → Use `useAssistantState(({ thread }) => thread.modelContext)`
- `useThreadComposer` → Use `useAssistantState(({ thread }) => thread.composer)`
- `useThreadList` → Use `useAssistantState(({ threads }) => threads)`

#### Migration Examples

**Before:**

```tsx
import {
  useThread,
  useThreadRuntime,
  useComposer,
  useComposerRuntime,
  useMessage,
} from "@assistant-ui/react";

function MyComponent() {
  // Reading state
  const messages = useThread((t) => t.messages);
  const isRunning = useThread((t) => t.isRunning);
  const composerText = useComposer((c) => c.text);
  const messageRole = useMessage((m) => m.role);

  // Using runtime for actions
  const threadRuntime = useThreadRuntime();
  const composerRuntime = useComposerRuntime();

  const handleSend = () => {
    composerRuntime.send();
  };

  const handleReload = () => {
    threadRuntime.reload();
  };

  return null;
}
```

**After:**

```tsx
import { useAssistantState, useAssistantApi } from "@assistant-ui/react";

function MyComponent() {
  // Reading state - all through single hook
  const messages = useAssistantState(({ thread }) => thread.messages);
  const isRunning = useAssistantState(({ thread }) => thread.isRunning);
  const composerText = useAssistantState(({ composer }) => composer.text);
  const messageRole = useAssistantState(({ message }) => message.role);

  // Using API for actions
  const { actions } = useAssistantApi();

  const handleSend = () => {
    actions.composer.send();
  };

  const handleReload = () => {
    actions.thread.reload();
  };

  return null;
}
```

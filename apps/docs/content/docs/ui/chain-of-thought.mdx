### Wire it into your assistant message

Mount `ChainOfThought` as the `components.ChainOfThought` slot on `MessagePrimitive.Parts`. The component subscribes to the chain-of-thought scope and renders all consecutive reasoning and tool-call parts inside a single collapsible panel.

```tsx title="/app/components/assistant-ui/thread.tsx"
import { MessagePrimitive } from "@assistant-ui/react";
import { ChainOfThought } from "@/components/assistant-ui/chain-of-thought"; // [!code ++]
import { MarkdownText } from "@/components/assistant-ui/markdown-text";
import { ToolFallback } from "@/components/assistant-ui/tool-fallback";

const AssistantMessage: FC = () => {
  return (
    <MessagePrimitive.Root>
      <MessagePrimitive.Parts
        components={{
          Text: MarkdownText,
          tools: { Fallback: ToolFallback },
          ChainOfThought, // [!code ++]
        }}
      />
    </MessagePrimitive.Root>
  );
};
```

The trigger shows a live activity label (e.g. "Searching web", "Reading sources") and an elapsed-time counter while streaming. Once the chain finishes, the panel auto-collapses back to a compact summary such as "Researched 3 sources (12s)".

  </Step>
</Steps>

## Customizing tool activity labels

Pass `toolActivityLabels` to override the label shown for a specific tool. The resolver receives the part status and can return a static string or compute one from the part itself.

```tsx
<ChainOfThought
  toolActivityLabels={{
    fetch_weather: ({ statusType }) =>
      statusType === "running" ? "Checking forecast..." : "Got forecast",
    create_pr: ({ statusType, fallbackLabel }) =>
      statusType === "incomplete" ? "PR failed" : fallbackLabel,
  }}
/>
```

If a tool has no entry, the component falls back to a humanized version of the tool name with verb inflection (e.g. `search_web` → "Searching web" / "Searched web").

Use the `"*"` catch-all key to localize or rewrite **every** tool's default label from a single resolver:

```tsx
<ChainOfThought
  toolActivityLabels={{
    "*": ({ fallbackLabel, statusType }) => translate(fallbackLabel, statusType),
  }}
/>
```

Each resolver receives the full `ToolActivityContext`:

| Field                                                  | Description                                                                  |
| ------------------------------------------------------ | ---------------------------------------------------------------------------- |
| `toolName`                                             | The tool's name (empty string for non-tool parts).                           |
| `statusType`                                           | Effective status: `"running"`, `"requires-action"`, `"complete"`, or `"incomplete"`. |
| `fallbackLabel`                                        | The humanized default label this resolver is overriding.                     |
| `partStatusType` / `chainStatusType` / `messageStatusType` | The granular underlying statuses, if you need them.                      |
| `part`                                                | The message part itself — compute a label from its arguments or result.      |

## Localization

Every user-facing string flows through a single `strings` seam (`ChainOfThoughtStrings`). The runtime `<ChainOfThought>` takes a `strings` prop; the composable `Trace` / `TraceDisclosure` read the same strings from `ChainOfThoughtStringsContext`. Switch languages below to see the panel update live:

<ChainOfThoughtLocalizationSample />

Override any subset — unspecified keys fall back to English. Duration- and count-dependent labels are functions so each locale controls its own grammar:

```tsx
<ChainOfThought
  strings={{
    reasoning: "Razonamiento",
    thinking: "Pensando...",
    reasoningHidden: "Razonamiento oculto.",
    reasoningActivity: (snippet) => `Pensando: ${snippet}`,
    jumpToLatest: "Más reciente",
    jumpToLatestLabel: "Ir a la última actualización",
    working: "Trabajando...",
    done: (s) => (s != null ? `Listo en ${s}s` : "Listo"),
    stopped: (s) => (s != null ? `Detenido tras ${s}s` : "Detenido"),
  }}
/>
```

The collapsed trace summary ("Researched 3 sources", "Ran 2 tools", "Stopped after 4 steps") is built by `traceSummary`, whose default pluralizes via `Intl.PluralRules`. Override it to control the grammar for locales with non-binary plural categories:

```tsx
const fr = new Intl.PluralRules("fr-FR");

<ChainOfThought
  strings={{
    traceSummary: ({ searchSteps, toolSteps, incomplete, durationSec }) => {
      const base = incomplete
        ? `Arrêté après ${toolSteps} ${fr.select(toolSteps) === "one" ? "étape" : "étapes"}`
        : `Consulté ${searchSteps} ${fr.select(searchSteps) === "one" ? "source" : "sources"}`;
      return durationSec != null ? `${base} (${durationSec}s)` : base;
    },
  }}
/>;
```

<Callout type="info">
  In the `traceSummary` stats, `searchSteps` and `toolSteps` are **disjoint** — a
  search counts only toward `searchSteps`, so `toolSteps` means "non-search tool
  calls". `totalSteps` counts leaf steps only; group nodes are not counted.
</Callout>

For the composable `Trace` / `TraceDisclosure` (which have no `strings` prop), provide the seam through context instead:

```tsx
import {
  ChainOfThought,
} from "@/components/assistant-ui/chain-of-thought";
import {
  ChainOfThoughtStringsContext,
  mergeChainOfThoughtStrings,
} from "@/components/assistant-ui/chain-of-thought/strings";

const es = mergeChainOfThoughtStrings({ reasoning: "Razonamiento" });

<ChainOfThoughtStringsContext.Provider value={es}>
  <ChainOfThought.TraceDisclosure trace={trace} />
</ChainOfThoughtStringsContext.Provider>;
```

<Callout type="info">
  `reasoning`, `working`, `traceSummary`, `done`, and `stopped` are rendered by
  the **trigger**, so they localize through `TraceDisclosure` (or your own
  `ChainOfThought.Trigger`). A bare `ChainOfThought.Trace` only renders your
  trace data plus the jump-to-latest control — both already read the same
  context, so wrapping any composition in the provider localizes every piece
  that consumes the seam.
</Callout>

Tool activity verbs are localized separately via the `toolActivityLabels` `"*"` resolver shown above. (`ChainOfThought.TraceDisclosure` also accepts a per-instance `summary` formatter that takes precedence over `traceSummary`.)

The panel is RTL-aware: the disclosure chevrons and the jump-to-latest button mirror automatically under `dir="rtl"`.

## Custom trigger

Render your own trigger body via `renderTriggerContent`. The resolver receives the resolved trigger state, including the localized `reasoningLabel` (so you don't have to hardcode English):

```tsx
<ChainOfThought
  renderTriggerContent={({
    phase,
    isOpen,
    reasoningLabel,
    activityLabel,
    elapsedSeconds,
  }) => (
    <span>
      {phase === "running" ? reasoningLabel : "Thought"}
      {elapsedSeconds ? ` for ${elapsedSeconds}s` : null}
      {activityLabel ? ` — ${activityLabel}` : null}
    </span>
  )}
/>
```

<Callout type="warn">
  A custom trigger body fully replaces the default — including the chevron.
  Include a visible text label (or an `aria-label`) so the toggle keeps an
  accessible name.
</Callout>

## Composing your own layout

`ChainOfThought.Root`, `.Trigger`, `.Content`, `.Timeline`, and `.Step*` primitives are exported for custom compositions. The `Root` owns the open state — pass `defaultOpen` (uncontrolled) or `open`/`onOpenChange` (controlled):

```tsx
import { ChainOfThought } from "@/components/assistant-ui/chain-of-thought";

<ChainOfThought.Root variant="muted" defaultOpen>
  <ChainOfThought.Trigger phase="complete" activityLabel="Researched 3 sources" />
  <ChainOfThought.Content>
    <ChainOfThought.Timeline>
      <ChainOfThought.Step status="complete" type="search">
        <ChainOfThought.StepHeader>Searched docs</ChainOfThought.StepHeader>
        <ChainOfThought.StepBody>3 results</ChainOfThought.StepBody>
      </ChainOfThought.Step>
    </ChainOfThought.Timeline>
  </ChainOfThought.Content>
</ChainOfThought.Root>;
```

<Callout type="info">
  The `Trigger`'s `isOpen` prop is only a fallback used when it renders outside a
  `Root`. Inside a `Root`, the resolved open state comes from the Root's
  disclosure context, so set the state on the `Root`.
</Callout>

## Rendering a pre-shaped trace

For agents that produce a structured trace server-side, render it without the runtime via `ChainOfThought.Trace` (an always-visible timeline you can embed anywhere) or `ChainOfThought.TraceDisclosure` (that timeline wrapped in the collapsible trigger shell). A trace is a `TraceNode[]` of steps and groups — groups can nest, carry a `variant: "subagent"`, and expand on click:

<ChainOfThoughtStructuredSample />

```tsx
import {
  ChainOfThought,
  type TraceNode,
} from "@/components/assistant-ui/chain-of-thought";

const trace: TraceNode[] = [
  {
    kind: "group",
    id: "research",
    label: "Research",
    status: "complete",
    children: [
      {
        kind: "step",
        id: "s1",
        label: "Searched docs",
        type: "search",
        toolName: "search_web",
        status: "complete",
      },
      { kind: "step", id: "s2", label: "Fetched 2 sources", status: "complete" },
    ],
  },
];

<ChainOfThought.TraceDisclosure trace={trace} />;
```

### Deriving a trace from message parts

Omit `trace` and `Trace` / `TraceDisclosure` derive the timeline from the active message's parts. Control how parts become steps with two overrides:

- **`groupingFunction`** — how parts are grouped into rows. The default groups consecutive parts by their `parentId`.
- **`inferStep`** — maps each group to a step's visual metadata (`{ label, type, status, stepLabel, icon }`).

```tsx
<ChainOfThought.TraceDisclosure
  inferStep={({ parts, isActive }) => {
    const tool = parts.find((p) => p.type === "tool-call");
    return {
      label: tool ? `Running ${tool.toolName}` : "Thinking",
      type: tool ? "tool" : "text",
      status: isActive ? "active" : "complete",
    };
  }}
/>
```

## API Reference

### `ChainOfThought` props

| Prop                     | Type                                       | Default     | Description                                                                                              |
| ------------------------ | ------------------------------------------ | ----------- | ------------------------------------------------------------------------------------------------------- |
| `constrainHeight`        | `boolean`                                  | `false`     | Caps the timeline height (scrollable) and shows a jump-to-latest affordance when scrolled up. The bottom fade gradient is always present. |
| `autoCollapseOnComplete` | `boolean`                                  | `true`      | Collapse the panel automatically once streaming completes.                                              |
| `toolActivityLabels`     | `Record<string, ToolActivity>`             | –           | Per-tool label resolver overrides (use the `"*"` key for a catch-all).                                  |
| `renderTriggerContent`   | `(state) => ReactNode`                     | –           | Render a custom trigger body. Receives `{ phase, isOpen, reasoningLabel, activityLabel, elapsedSeconds }`. |
| `variant`                | `"ghost" \| "outline" \| "muted"`          | `"ghost"`   | Visual chrome.                                                                                           |
| `strings`                | `Partial<ChainOfThoughtStrings>`           | –           | Override built-in strings for localization. See [Localization](#localization).                          |

### `ChainOfThought.Trace` / `.TraceDisclosure` props

Both accept a pre-shaped `trace` **or** derive one from message parts when `trace` is omitted.

| Prop                | Type                              | Default               | Description                                                              |
| ------------------- | --------------------------------- | --------------------- | ------------------------------------------------------------------------ |
| `trace`             | `TraceNode[]`                     | –                     | Pre-shaped trace. Omit to derive from the active message's parts.        |
| `maxDepth`          | `number`                          | `2`                   | How deep nested groups can expand.                                       |
| `allowGroupExpand`  | `boolean`                         | `true`                | Whether group rows are expandable.                                       |
| `nodeComponents`    | `{ GroupSummary?, StepBody? }`    | –                     | Override slots for the group summary and step body renderers.            |
| `constrainHeight`   | `boolean`                         | `true`                | Cap the timeline height and scroll. (Note: the runtime `ChainOfThought` defaults this to `false`.) |
| `groupingFunction`  | `(parts) => Group[]`              | groups by `parentId`  | _Parts-derived only._ How message parts are grouped into steps.          |
| `inferStep`         | `(args) => TraceStepMeta`         | `defaultInferStep`    | _Parts-derived only._ Maps each group to `{ label, type, status, stepLabel, icon }`. |

`TraceDisclosure` adds the collapsible shell props:

| Prop                                   | Type                              | Default            | Description                                                       |
| -------------------------------------- | --------------------------------- | ------------------ | ----------------------------------------------------------------- |
| `label`                                | `string`                          | `strings.working`  | Trigger label shown while the chain is streaming.                 |
| `summary`                              | `string \| TraceSummaryFormatter` | `strings.traceSummary` | Collapsed summary; takes precedence over the `traceSummary` string. |
| `autoCollapseOnComplete`               | `boolean`                         | `true`             | Collapse when streaming ends.                                     |
| `disableGroupExpansionWhileStreaming`  | `boolean`                         | `true`             | Lock group expansion until streaming finishes.                   |
| `rootProps` / `triggerProps` / `contentProps` | `object`                   | –                  | Escape hatches forwarded to the underlying Root / Trigger / Content (e.g. `rootProps={{ variant }}`). |

| `ChainOfThought.Step`            | A single step. Props: `status` (`pending`/`active`/`complete`/`error`), `active`, `type`, `stepLabel`, `icon`, `error` + `onRetry` (renders an inline error row with a Retry button), `iconPulse`. |

| `ChainOfThought.Trace`           | Render a `TraceNode[]` or derive one from message parts.                                                                     |

### `ChainOfThoughtStrings`

The full localization seam. Override any subset via the `strings` prop (runtime) or `ChainOfThoughtStringsContext` (composable).

| Key                 | Type                            | Default                                            |
| ------------------- | ------------------------------- | -------------------------------------------------- |
| `reasoning`         | `string`                        | `"Reasoning"`                                      |
| `thinking`          | `string`                        | `"Thinking..."`                                    |
| `reasoningHidden`   | `string`                        | `"Reasoning hidden."`                              |
| `reasoningActivity` | `(snippet: string) => string`   | `` snippet => `Thinking: ${snippet}` ``            |
| `jumpToLatest`      | `string`                        | `"Latest"`                                         |
| `jumpToLatestLabel` | `string`                        | `"Jump to latest chain-of-thought update"` (`aria-label`) |
| `working`           | `string`                        | `"Working..."`                                     |
| `done`              | `(seconds?: number) => string`  | `` s => s != null ? `Done in ${s}s` : "Done" ``    |
| `stopped`           | `(seconds?: number) => string`  | `` s => s != null ? `Stopped after ${s}s` : "Stopped" `` |
| `traceSummary`      | `(stats) => string`             | Pluralized via `Intl.PluralRules`                  |

`traceSummary` receives `{ totalSteps, searchSteps, toolSteps, incomplete, durationSec? }`.

### `TraceNode` model

A `TraceNode` is either a `TraceStep` (leaf) or a `TraceGroup` (nested).

**`TraceStep`** — `{ kind: "step", id, label?, stepLabel?, type?, icon?, status?, toolName?, detail?, output?, meta? }`. `status` is a `TraceStatus`: `"pending" | "running" | "complete" | "incomplete"`.

**`TraceGroup`** — `{ kind: "group", id, label, status?, summary?, children, variant?, meta? }`. Set `variant: "subagent"` for the bot-styled subagent chrome.

The `type` field selects the step icon (`StepType`):

| `StepType`              | Icon          |
| ----------------------- | ------------- |
| `search`                | Magnifier     |
| `web_search`            | Globe         |
| `image`                 | Image         |
| `text` / `write`        | File          |
| `code` / `javascript`   | Terminal      |
| `merge`                 | Git merge     |
| `tool`                  | Wrench        |
| `complete`              | Check circle  |
| `error`                 | Alert circle  |
| `default`               | Bullet dot    |

### Use it with grouped message parts

For new message UIs, use `MessagePrimitive.GroupedParts` and compose the
Chain of Thought primitives around the grouped reasoning and tool-call parts.

```tsx title="/components/assistant-ui/thread.tsx"
import { MessagePrimitive, groupPartByType } from "@assistant-ui/react";
import { MarkdownText } from "@/components/assistant-ui/markdown-text";
import { ChainOfThought } from "@/components/assistant-ui/chain-of-thought";
import { ToolFallback } from "@/components/assistant-ui/tool-fallback";

<MessagePrimitive.GroupedParts
  groupBy={groupPartByType({
    reasoning: ["group-chain-of-thought"],
    "tool-call": ["group-chain-of-thought"],
    "standalone-tool-call": [],
  })}
>
  {({ part, children }) => {
    switch (part.type) {
      case "group-chain-of-thought": {
        const phase =
          part.status.type === "running"
            ? "running"
            : part.status.type === "requires-action"
              ? "requires-action"
              : part.status.type === "incomplete"
                ? "incomplete"
                : "complete";
        const running = part.status.type === "running";
        return (
          <ChainOfThought.Root defaultOpen={running}>
            <ChainOfThought.Trigger phase={phase} />
            <ChainOfThought.Content aria-busy={running}>
              <ChainOfThought.Timeline autoScroll={running}>
                {children}
              </ChainOfThought.Timeline>
            </ChainOfThought.Content>
          </ChainOfThought.Root>
        );
      }
      case "reasoning":
        return (
          <ChainOfThought.Step type="text" status="complete">
            <ChainOfThought.StepBody>
              <MarkdownText />
            </ChainOfThought.StepBody>
          </ChainOfThought.Step>
        );
      case "tool-call":
        return (
          <ChainOfThought.Step
            type="tool"
            status={
              part.status.type === "running" ||
              part.status.type === "requires-action"
                ? "active"
                : part.status.type === "incomplete"
                  ? "error"
                  : "complete"
            }
            active={
              part.status.type === "running" ||
              part.status.type === "requires-action"
            }
          >
            <ChainOfThought.StepBody>
              {part.toolUI ?? <ToolFallback {...part} />}
            </ChainOfThought.StepBody>
          </ChainOfThought.Step>
        );
      case "text":
        return <MarkdownText />;
      default:
        return null;
    }
  }}
</MessagePrimitive.GroupedParts>
```

  </Step>
</Steps>

## Customizing Tool Activity Labels

The runtime `<ChainOfThought />` component can turn tool names into readable
activity labels. Override a specific tool by name, or provide `"*"` as a
fallback resolver:

```tsx
<ChainOfThought
  toolActivityLabels={{
    web_search: ({ statusType }) =>
      statusType === "running" ? "Searching docs" : "Searched docs",
    "*": ({ fallbackLabel, statusType }) =>
      statusType === "running"
        ? `Using ${fallbackLabel}`
        : `Used ${fallbackLabel}`,
  }}
/>
```

## Localization

Pass `strings` to the runtime component, or pass labels directly when composing
the primitives.

<ChainOfThoughtLocalizationSample />

```tsx
<ChainOfThought
  strings={{
    reasoning: "Razonamiento",
    thinking: "Pensando...",
    done: (seconds) => (seconds ? `Listo en ${seconds}s` : "Listo"),
    stopped: (seconds) =>
      seconds ? `Detenido tras ${seconds}s` : "Detenido",
  }}
/>
```

## Custom Trigger

Use `renderTriggerContent` when the default trigger copy is not enough:

```tsx
<ChainOfThought
  renderTriggerContent={({ phase, activityLabel, elapsedSeconds }) => (
    <span>
      {phase === "running" ? "Working" : "Reasoning"}
      {activityLabel ? ` - ${activityLabel}` : null}
      {elapsedSeconds ? ` (${elapsedSeconds}s)` : null}
    </span>
  )}
/>
```

## API Reference

### `ChainOfThought` Props

| Prop                     | Type                                      | Default | Description |
| ------------------------ | ----------------------------------------- | ------- | ----------- |
| `constrainHeight`        | `boolean`                                 | `false` | Caps the timeline height and scrolls within the panel. |
| `toolActivityLabels`     | `Record<string, ToolActivity>`            | -       | Overrides generated labels for tool-call activity. |
| `renderTriggerContent`   | `(args) => ReactNode`                     | -       | Replaces the default trigger content. |
| `autoCollapseOnComplete` | `boolean`                                 | `true`  | Collapses after streaming completes. |
| `variant`                | `"ghost" \| "muted" \| "outline"`        | `"ghost"` | Visual chrome for the root panel. |
| `strings`                | `Partial<ChainOfThoughtStrings>`          | -       | Overrides built-in strings. |

### Composable API

| Component | Description |
| --------- | ----------- |
| `ChainOfThought.Root` | Collapsible container with scroll lock and variants. |
| `ChainOfThought.Trigger` | Button with phase-aware label, activity, elapsed time, and chevron. |
| `ChainOfThought.Content` | Animated collapsible content wrapper. |
| `ChainOfThought.Placeholder` | Empty-state placeholder. |
| `ChainOfThought.Timeline` | Vertical list with optional auto-scroll and jump-to-latest control. |
| `ChainOfThought.Step` | A timeline item with status, icon, label, and body slots. |
| `ChainOfThought.StepHeader` | Label slot for a step. |
| `ChainOfThought.StepBody` | Body slot for a step. |

## Agent Traces

Pre-shaped server-side agent traces are intentionally outside this component's
initial API. They should be introduced as a separate `AgentTrace` component so
the Chain of Thought component can stay focused on message-scoped reasoning and
tool activity.

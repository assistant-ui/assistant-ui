---
title: Generative UI
description: Render agent-described React UI from OpenUI Lang with a consumer-defined component library.
---

`MessagePrimitive.GenerativeUI` renders UI described by the agent at runtime
using [OpenUI Lang](https://www.openui.com/docs/openui-lang). The agent emits
a `generative-ui` message part with a `source` string (OpenUI Lang). On web,
`@assistant-ui/react` wraps `@openuidev/react-lang` `Renderer` and your
**consumer-defined library** (Zod schemas are the security boundary).

> **Opt-in feature:** The default shadcn `Thread` does **not** render
> `generative-ui` parts. Wire the primitive explicitly — see
> [Opt-in wiring](#opt-in-wiring).

> **Web only:** OpenUI rendering lives in `@assistant-ui/react`. React Native
> and Ink re-export core types only.

## Which generative UI pattern?

| Pattern | API | Best for | Streaming |
|---------|-----|----------|-----------|
| **Generative UI primitive** | `MessagePrimitive.GenerativeUI` + OpenUI library | Dashboards, cards, layouts from your component vocabulary | Append `source` incrementally; `isStreaming` enables progressive parse |
| **Tool UI** | `makeAssistantToolUI` | Interactive widgets tied to a known tool | Tool **args** stream |
| **LangGraph data UI** | `makeAssistantDataUI` + `ui_message` | LangGraph agents emitting UI on the custom channel | UI messages on LangGraph stream |

See also: [Tool UI guide](/docs/guides/tool-ui), [LangGraph generative UI](/docs/runtimes/langgraph/generative-ui).

## When not to use the primitive

- **User input and two-way interaction** → [Tool UI](/docs/guides/tool-ui) or [Interactables](/docs/guides/interactables)
- **LangGraph `push_ui_message`** → [LangGraph data UI](/docs/runtimes/langgraph/generative-ui)
- **Untrusted HTML or arbitrary code** → [Artifacts](/docs/guides/artifacts) with sandboxed frames

## Quick start

### 1. Define an OpenUI library

```tsx title="components/gui/library.tsx"
import { defineComponent, createLibrary } from "@assistant-ui/react";
import { z } from "zod/v4";

const Card = defineComponent({
  name: "Card",
  description: "Display card",
  props: z.object({ title: z.string().optional() }),
  component: ({ props, renderNode, children }) => (
    <div className="rounded-xl border p-4">
      <div className="font-semibold">{props.title}</div>
      {children?.map((c, i) => <span key={i}>{renderNode(c)}</span>)}
    </div>
  ),
});

export const myLibrary = createLibrary({
  root: "Card",
  components: [Card /* Button, Stack, ... */],
});
```

Use `myLibrary.prompt()` in your system prompt so the model knows how to emit OpenUI Lang.

### 2. Wire the primitive

```tsx
<MessagePrimitive.GenerativeUI library={myLibrary} />
```

Or pass `library` to `MessagePrimitive.Parts` for a default slot:

```tsx
<MessagePrimitive.Parts library={myLibrary} />
```

### 3. Have the agent emit UI

**ExternalStore / manual messages:**

```ts
{ type: "generative-ui", source: 'root = Card([], "Hello")' }
```

**AI SDK (`useChatRuntime`)** — stream OpenUI Lang in assistant text and split at render time with `extractOpenUILangFromText` from `@assistant-ui/react-ai-sdk`, or append native `generative-ui` parts from your adapter.

Live examples: [`examples/with-generative-ui`](https://github.com/assistant-ui/assistant-ui/tree/main/examples/with-generative-ui) — Tool UI (`/`), static OpenUI Lang (`/primitive`), streaming GUI chat (`/gui-chat`).

## Opt-in wiring

### Pattern 1 — `MessagePrimitive.Parts` slot

```tsx
<MessagePrimitive.Parts
  components={{
    generativeUI: () => (
      <MessagePrimitive.GenerativeUI library={myLibrary} />
    ),
  }}
/>
```

### Pattern 2 — `GroupedParts` case

```tsx
case "generative-ui":
  return <MessagePrimitive.GenerativeUI library={myLibrary} />;
case "text": {
  const extracted = extractOpenUILangFromText(part.text);
  if (extracted) {
    return (
      <>
        {extracted.remainder ? <MarkdownText /> : null}
        <MessagePrimitive.GenerativeUI
          library={myLibrary}
          source={extracted.source}
        />
      </>
    );
  }
  return <MarkdownText />;
}
```

Exclude `render_gui` from tool-group `groupBy` if you keep the legacy JSON bridge.

## Message part shape

```ts
type GenerativeUIMessagePart = {
  type: "generative-ui";
  source: string; // raw OpenUI Lang — append/stream incrementally
  id?: string;
};
```

## Streaming

Append to `source` as tokens arrive. Pass `isStreaming` (defaults from message
status) so the OpenUI parser handles partial definitions gracefully.

With AI SDK, include `library.prompt()` in the system prompt and stream OpenUI
Lang as assistant content. Use `extractOpenUILangFromText` to split text parts
at render time.

## Security

Your OpenUI library Zod schemas define which components exist and which props
are valid. Untrusted agent output is parsed against those schemas — do not
register components that execute arbitrary props (e.g. `dangerouslySetInnerHTML`).

## Limitations

- **Assistant Cloud / `aui/v0`:** `generative-ui` encode not supported this release
- **LangChain `useStreamRuntime`:** append throws for `generative-ui` — use ExternalStore or manual parts
- **React Native / Ink:** types only; use web renderer or a custom slot

## Migration from JSON spec (0.2.x)

Removed exports: `GenerativeUISpec`, `GenerativeUINode`, `GenerativeUIRender`,
`GenerativeUIComponentRegistry`, `GenerativeUIRenderError`.

`MessagePrimitive.Parts` config changed:

```tsx
// Before
components={{ generativeUI: { components: { Card }, Fallback } }}

// After
components={{ generativeUI: () => <MessagePrimitive.GenerativeUI library={lib} /> }}
```

Replace JSON `spec` parts with OpenUI Lang `source` strings.

## Composing with other primitives

`generative-ui` is a regular message part type — compose with text, tools, and
reasoning in the same message via `MessagePrimitive.Parts` or `GroupedParts`.

---
title: Message Editing
description: Allow users to edit their messages with custom editor interfaces.
---

Give users the ability to edit their sent messages. When a user edits a message, the conversation is resubmitted from that point and a new branch is created.

## Quick Start

Add an edit composer to your Thread component by using `ComposerPrimitive` and `ActionBarPrimitive.Edit`:

```tsx
import {
  ActionBarPrimitive,
  ComposerPrimitive,
  MessagePrimitive,
  ThreadPrimitive,
} from "@assistant-ui/react";

// 1. Add EditComposer to Thread messages
const Thread = () => (
  <ThreadPrimitive.Root>
    <ThreadPrimitive.Viewport>
      <ThreadPrimitive.Messages
        components={{
          UserMessage,
          EditComposer, // rendered while message is in edit mode
        }}
      />
    </ThreadPrimitive.Viewport>
  </ThreadPrimitive.Root>
);

// 2. Add an Edit button to user messages
const UserMessage = () => (
  <MessagePrimitive.Root>
    <MessagePrimitive.Parts />
    <ActionBarPrimitive.Root>
      <ActionBarPrimitive.Edit />
    </ActionBarPrimitive.Root>
  </MessagePrimitive.Root>
);

// 3. Define the EditComposer component
const EditComposer = () => (
  <MessagePrimitive.Root>
    <ComposerPrimitive.Root>
      <ComposerPrimitive.Input />
      <ComposerPrimitive.Cancel />
      <ComposerPrimitive.Send />
    </ComposerPrimitive.Root>
  </MessagePrimitive.Root>
);
```

## Enabling Editing in ExternalStoreRuntime

With `ExternalStoreRuntime`, provide an `onEdit` callback to handle edited messages:

```tsx
import { useExternalStoreRuntime, type EditComposerAppendMessage } from "@assistant-ui/react";

const runtime = useExternalStoreRuntime({
  messages,
  setMessages,
  onNew: async (message) => {
    // handle new messages
  },
  onEdit: async (message) => {
    // message.parentId is the ID of the message being edited
    // Truncate messages up to the edited message and resubmit
    const parentIndex = messages.findIndex((m) => m.id === message.parentId);
    const truncated = messages.slice(0, parentIndex + 1);

    const userMessage = { role: "user" as const, content: message.content };
    setMessages([...truncated, userMessage]);

    const response = await callBackend(message.content);
    setMessages((prev) => [...prev, response]);
  },
});
```

## Auto-Resend After Edit

Editing a message automatically resends the conversation when the user clicks Send in the `EditComposer`. The runtime handles branching — the old continuation becomes an alternate branch and the new one becomes active.

If you want to trigger an edit **programmatically** (e.g., from a suggestion or while the thread is running), use the Context API:

```tsx
import { useAui, useRef } from "react";
import { useAuiEvent } from "@assistant-ui/react";

function SmartEditButton({ newText }: { newText: string }) {
  const aui = useAui();
  const resendRef = useRef(false);

  // After cancellation, auto-send the staged text
  useAuiEvent("thread.runEnd", () => {
    if (!resendRef.current) return;
    resendRef.current = false;
    aui.composer().send();
  });

  const handleClick = () => {
    const isRunning = aui.thread().getState().isRunning;
    aui.composer().setText(newText);

    if (isRunning) {
      resendRef.current = true;
      aui.composer().cancel(); // runEnd handler will send
    } else {
      aui.composer().send();
    }
  };

  return <button onClick={handleClick}>Use This Text</button>;
}
```

## Styling the Edit Mode

The edit composer is only rendered when a message is in edit mode. Use `MessagePrimitive.If` to control visibility of other elements:

```tsx
const UserMessage = () => (
  <MessagePrimitive.Root>
    {/* Hide content when in edit mode */}
    <MessagePrimitive.If editing={false}>
      <MessagePrimitive.Parts />
      <ActionBarPrimitive.Root>
        <ActionBarPrimitive.Edit />
        <ActionBarPrimitive.Copy />
      </ActionBarPrimitive.Root>
    </MessagePrimitive.If>
  </MessagePrimitive.Root>
);
```

## API Reference

### ActionBarPrimitive.Edit

A button that enters edit mode for the current message. Only renders when editing is available.

### ComposerPrimitive.Cancel

Exits edit mode without submitting. Returns the message to its original state.

### ComposerPrimitive.Send

Submits the edited message, creating a new branch from the edit point.

### onEdit (ExternalStoreRuntime)

```ts
onEdit?: (message: EditComposerAppendMessage) => Promise<void>
```

Called when a user submits an edited message. The `message.parentId` field identifies which message was edited. If not provided, the edit button will not appear.

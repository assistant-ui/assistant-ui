## Install

```sh
npm install @assistant-ui/tap
```

## Wire the alias

Inside a resource you import hooks from `"react"`, but they need to run as tap's
hooks instead of React's. A build alias rewrites the bare `react` specifier to
tap's drop-in (`@assistant-ui/tap/react-shim`) for code that runs inside a
resource, while leaving real React components untouched.

Use the integration for your bundler:

```ts title="vite.config.ts"
import { aui } from "@assistant-ui/vite";

export default defineConfig({
  plugins: [aui()],
});
```

```ts title="next.config.ts"
import { withAui } from "@assistant-ui/next";

export default withAui({
  // your Next.js config
});
```

```ts title="metro.config.js"
const { withAui } = require("@assistant-ui/metro");

module.exports = withAui(getDefaultConfig(__dirname));
```

<Callout type="info">
If you already use `@assistant-ui/store` (or any assistant-ui distribution) with
one of these integrations, the alias is wired for you. Skip this step.

Not using a supported bundler? Alias the bare `react` specifier to
`@assistant-ui/tap/react-shim` yourself, making sure the rewrite does not apply to
tap's own files.
</Callout>

## Define a resource

A resource is a React component without the UI. Author it with `resource` and a
named function, and use the hooks you already know, imported from `"react"`.

```ts
import { resource } from "@assistant-ui/tap";
import { useState, useEffect } from "react";

const Counter = resource(function Counter() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    console.log(`Count: ${count}`);
  }, [count]);

  return { count, increment: () => setCount((c) => c + 1) };
});
```

Instead of JSX, a resource returns a plain value. That return value is its public
API: the state and methods you want callers to use.

<Callout type="warn">
One rule that differs from React: you cannot call a setter during render. To
adjust state in response to props, derive it during render instead. See
[How tap differs from React](/tap/docs/differences-from-react).
</Callout>

## Use it in React

`useResource` hosts a resource inside a React component. The component re-renders
whenever the resource's state changes, and unmounts the resource when it unmounts.

```tsx
import { useResource } from "@assistant-ui/tap";

function CounterButton() {
  const { count, increment } = useResource(Counter());

  return <button onClick={increment}>Count: {count}</button>;
}
```

Calling the factory (`Counter()`) creates an inert element; `useResource` brings
it to life.

## Compose resources

Resources host other resources with the same `useResource` hook. Unlike React, a
parent reads its child's return value directly, so composition builds up state and
logic, not just trees.

```ts
import { resource, useResource } from "@assistant-ui/tap";
import { useEffect } from "react";

const AutoCounter = resource(function AutoCounter() {
  const counter = useResource(Counter());

  useEffect(() => {
    const id = setInterval(counter.increment, 1000);
    return () => clearInterval(id);
  }, [counter]);

  return { count: counter.count };
});
```

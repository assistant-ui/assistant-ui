Inside a resource you use React's hooks, imported from `"react"`: `useState`,
`useReducer`, `useEffect`, `useMemo`, `useCallback`, `useRef`, `useEffectEvent`,
and `use`. They behave the way they do in a component, with a couple of small
[differences](/tap/docs/differences-from-react), and the same rules of hooks
apply.

## Rules of hooks

<div className="not-prose flex flex-col gap-1.5 text-sm">
<p><span className="text-emerald-400/80">&#x2713;</span> Call hooks at the top level of a resource body</p>
<p><span className="text-emerald-400/80">&#x2713;</span> Call hooks at the top level of a custom `use*` hook</p>
<p><span className="text-rose-400/80">&#x2717;</span> **Don't** call hooks in conditions, loops, or nested functions</p>
<p><span className="text-rose-400/80">&#x2717;</span> **Don't** call hooks in event handlers or `try`/`catch`/`finally`</p>
<p><span className="text-rose-400/80">&#x2717;</span> **Don't** call hooks in functions passed to `useState`, `useMemo`, or `useEffect`</p>
</div>

## State

```ts
import { useState, useReducer } from "react";

const [count, setCount] = useState(0);

// with a lazy initializer
const [data, setData] = useState(() => expensiveComputation());

setCount(5);
setCount((prev) => prev + 1);
```

Unlike React, calling a setter (or a reducer's `dispatch`) during render throws.
To adjust state in response to props, derive it during render, or use
[`useReducerWithDerivedState`](#usereducerwithderivedstate).

## useReducerWithDerivedState

<Callout type="warn">
Experimental, and tap-only (import it from `@assistant-ui/tap` and call it inside
a resource render). May change.
</Callout>

Like `useReducer`, but with a `getDerivedState` function that runs during render
after processing queued actions. It serves the same use case as React's
[`getDerivedStateFromProps`](https://react.dev/reference/react/Component#static-getderivedstatefromprops):
adjusting internal state in response to changing props without calling
`set`/`dispatch` during render. If `getDerivedState` returns a new value (by
`Object.is`), that value becomes the current state and triggers a re-render.

```ts
import { useReducerWithDerivedState } from "@assistant-ui/tap";

type State = { items: string[]; selectedId: string | null; prevItems: string[] };
type Action = { type: "select"; id: string };

const [state, dispatch] = useReducerWithDerivedState(
  (state: State, action: Action): State => {
    switch (action.type) {
      case "select":
        return { ...state, selectedId: action.id };
    }
  },
  // reset selection when items change (getDerivedStateFromProps pattern)
  (state) => {
    if (state.prevItems === items) return state;
    return { ...state, items, prevItems: items, selectedId: null };
  },
  { items, selectedId: null, prevItems: items },
);
```

## Effects

```ts
import { useEffect } from "react";

useEffect(() => {
  const ws = new WebSocket("ws://localhost:8080");
  ws.onmessage = (event) => setMessages((prev) => [...prev, event.data]);
  return () => ws.close();
}, []);
```

Without a dependency array, the effect runs after every render. tap has a single
effect primitive, so `useLayoutEffect` behaves the same as `useEffect`.

## useEffectEvent

A stable function that always calls the latest closure. Useful for event handlers
passed to effects.

```ts
import { useEffect, useEffectEvent } from "react";

const onMessage = useEffectEvent((msg: string) => {
  setMessages((prev) => [...prev, msg]); // always the latest state
});

useEffect(() => {
  socket.addEventListener("message", onMessage);
  return () => socket.removeEventListener("message", onMessage);
}, [socket]);
```

## Memoized and stable values

```ts
import { useMemo, useCallback, useRef } from "react";

const sorted = useMemo(() => items.toSorted((a, b) => a.name.localeCompare(b.name)), [items]);
const handleClick = useCallback(() => setCount((c) => c + 1), []);
const ref = useRef(0);
```

These behave exactly as they do in React. For a value that must be created
exactly once (a stable instance), use a lazy `useState` initializer:

```ts
const [emitter] = useState(() => new EventEmitter());
```

## Composition hooks

Resources can render other resources. These hooks (exported from
`@assistant-ui/tap`) manage child resource lifecycles, mounting, updating, and
cleaning up automatically:

- [`useResource`](/tap/docs/composition#useresource) renders a single child resource
- [`useResources`](/tap/docs/composition#useresources) renders a keyed list of child resources
- [`useResourceRoot`](/tap/docs/composition#useresourceroot) wraps a child resource with `getValue` / `subscribe`

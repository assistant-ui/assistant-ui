add `ActionRegistry` and wire `$action` dispatch end to end. `createActionRegistry(handlers)` builds the host-provided map from `$action.type` to `({ payload }) => unknown | Promise<unknown>`; `emptyActionRegistry` is an explicit no-op registry a host can pass. the renderer injects a `$dispatch` prop (alongside `$status`/`$action`) into interactive components' `render`, threaded from `JSONGenerativeUIOptions.actions` through the render context. omit `actions` for a read-only render: `$dispatch` is left un-injected, so interactive clicks stay silent (no handler call, no dev warning). the vocabulary's `Button`/`Select`/`Input`/`DatePicker` attach real event handlers (`onClick`/`onChange`/`onKeyDown`) that call `$dispatch($action)`, merging the user's runtime value (selected option, entered text, picked date) into the payload under the reserved `$input` key so a model-supplied `value` is never clobbered. an unknown action type degrades to a no-op with a dev warning rather than throwing. HITL resume-value typing is left as `unknown` (IR doc open question #2); the resume value reaching the runtime is a follow-up.
